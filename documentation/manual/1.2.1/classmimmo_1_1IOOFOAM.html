Detailed Description

IOOFOAM is the class to import/export and modify an OpenFOAM mesh in format of surface vtk Polydata for surface meshes and points cloud for volume mesh.

The surfaces patch (boundary of volume mesh) have to be imported as .vtk files, while the volume mesh has to be read as points file with OpenFOAM format.

When imported a triangulation filter is applied to the each polymesh of surface meshes (input of multiple patches allowed) and insert together in a unique pointer to MimmoObject objects (locally instantiated).

A scalar field related to nodes of each surface mesh is read if it is present. No other data location are available for now.

The volume points cloud (import of one "points" file allowed) is insert in pointer to a MimmoObject (locally instantiated).

Dependencies : vtk libraries (tested with vtk DataFile Version 4.0).

The xml available parameters, sections and subsections are the following :

Inherited from BaseManipulation:

  • ClassName: name of the class as mimmo.IOOFOAM;
  • Priority: uint marking priority in multi-chain execution;

Proper of the class:

  • ReadFlag: activate reading mode boolean;
  • VTKReadDirs: VTK reading directories path; there can be more than one with the following sub-structure :
    <VTKReadDirs>
    <dir0>
    <dir> path to the directory of VTK file </dir>
    </dir0>
    <dir1>
    <dir> path to the directory of VTK file </dir>
    </dir1>
    ...
    ...
    </VTKReadDirs>
  • VTKReadFilenames: names of files VTK for reading there can be more than one with the following sub-structure :
    <VTKReadFilenames>
    <file0>
    <filename> path to the directory of VTK file </dir>
    </file0>
    <file1>
    <filename> path to the directory of VTK file </dir>
    </file1>
    ...
    ...
    </VTKReadFilenames>
  • PointsReadDir: points file reading directory path (there can be only one);
  • PointsReadFilename: name of points file for reading (there can be only one);
  • WriteFlag: activate writing mode boolean;
  • VTKWriteDir: VTK file (one in output) writing directory path;
  • VTKWriteFilename: VTK name of file for writing;
  • Scalar: value to scale the scalar field eventually present on volume mesh (defined on points) [default 1.0];
  • Normalize: bool to define if the scalar field has to be normalize after read [default false].

In case of writing mode Geometries have to be mandatorily passed by port.
